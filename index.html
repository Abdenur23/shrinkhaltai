FAQ

Everything you need to know about ShrinkHalt

How accurate is the AI across different store layouts?
Our model was trained with geometric transformations, view‑aware kinematic features, and a FiLM‑conditioned TCN, achieving 96.6% weighted F1 on a scene‑held‑out test set. Performance generalizes across moderate viewpoint shifts, although extreme occlusion may reduce precision.
What about privacy and data retention?
The system processes pose keypoints, not raw video, at the edge. Only short anonymized clips of flagged events are shared with our remote review team. ShrinkHalt is SOC 2 type II compliant and never uses customer data for model training without explicit consent.
Does ShrinkHalt require new cameras?
No. It works with standard IP security cameras. You only need a low‑cost edge device (e.g., Jetson Orin Nano) per store section or every 4–6 cameras. Existing infrastructure is fully compatible.
How quickly can we deploy?
Most retailers deploy ShrinkHalt within 1–2 weeks. The edge software is pre‑configured, and remote reviewer onboarding happens in parallel. No lengthy data science or custom model training required.
What actions does the system detect?
Four operational macro‑classes: Floor Activity (normal browsing/picking), Consumption, Concealment in Bag (front/side bag), and On‑Body Concealment (pockets, hoodie, pants, outerwear). The taxonomy was optimized for loss prevention triage.
Is remote human review available 24/7?
Yes, our remote loss prevention team operates across multiple time zones. Average verification latency is under 15 seconds, with priority alerts for high‑risk behaviours.
Benchmark‑informed design

The core detection engine is a Temporal Convolutional Network (824K parameters) with FiLM conditioning, achieving 0.9661 weighted F1 on a challenging held‑out test set (27.5k real‑world pose sequences). Feature engineering uses skeleton‑based kinematics with view‑adaptive modulation, enabling robust edge deployment on low‑cost hardware. (Reference: Edge‑Native Action Triage, 2025)
